# Add a resource end to end

Use this guide when replacing the worked examples with your real domain or
adding another resource beside them. The pattern stays explicit: migration,
domain queries, API handlers, OpenAPI registration, generated TypeScript,
frontend page, route, and tests.

The fastest path is to copy `items` for simple CRUD or `posts` for lifecycle
transitions, filtered lists, pagination, get-by-id, and stats.

## Files you usually touch

For a resource named `widgets`:

```text
migrations/<timestamp>_create_widgets.sql
src/widgets.rs
src/lib.rs
src/api/widgets.rs
src/api/mod.rs
tests/api.rs
interface/src/api/schema.d.ts        # generated by just typegen
interface/src/pages/Widgets.tsx
interface/src/router.tsx
```

## Checklist

### 1. Add an append-only migration

Create a new migration whose timestamp sorts after every existing migration:

```text
migrations/<YYYYMMDDHHMMSS>_<description>.sql
```

Conventions used by the template:

- `TEXT` UUID primary keys generated in Rust.
- `INTEGER` booleans.
- `TEXT` timestamps.
- Indexes on sort/filter columns.

Never edit or rename a committed migration. sqlx records migration checksums,
so changing history can break existing databases.

### 2. Add the domain module

Create `src/<resource>.rs` with:

- row structs deriving `Debug`, `Serialize`, `sqlx::FromRow`, and `utoipa::ToSchema`;
- request payload structs deriving `Debug`, `Deserialize`, and `ToSchema`;
- plain async query functions that take `&SqlitePool`;
- by-id mutations returning `Result<bool, sqlx::Error>` from `rows_affected()`
  when callers map missing rows to 404.

Avoid repository traits or generic service layers until repeated real apps prove they are needed.

### 3. Export the domain module

Add `pub mod <resource>;` in `src/lib.rs`.

Only extend `AppState` for genuinely shared values. Most resources only need the existing `pool`.

### 4. Add API handlers

Create `src/api/<resource>.rs` with thin axum handlers:

- accept `State(state): State<AppState>`;
- return `Result<_, AppError>`;
- validate request input in the handler and return `AppError::BadRequest` for invalid input;
- map `None` or `false` domain results to `AppError::NotFound`;
- return `StatusCode::CREATED` with `Json` for create handlers;
- return `StatusCode::NO_CONTENT` for delete-like handlers.

Every handler must have a `#[utoipa::path(...)]` annotation with:

- the full literal path including `/api`;
- a per-resource `tag`;
- `params` for path/query inputs;
- `request_body` for POST/PUT/PATCH bodies;
- every response status with a `body =` schema when a body exists.

### 5. Register the API in three places

In `src/api/mod.rs`:

1. Add `pub mod <resource>;`.
2. Add every handler to the `#[openapi(paths(...))]` list and every new schema to `components(schemas(...))`.
3. Add `.route("/api/...", ...)` entries before `.fallback(crate::frontend::spa)`.

Use axum 0.8 brace syntax for route params, such as `{id}`.

Most common mistake: an endpoint missing from OpenAPI `paths(...)` still works
at runtime but disappears from `interface/src/api/schema.d.ts` after typegen.

### 6. Add backend tests

Extend `tests/api.rs` using `test_app()` and `body_json()`.

For each resource, cover at least:

- one happy-path roundtrip;
- one invalid-input 400;
- one missing-record 404.

Run:

```bash
just test
```

### 7. Regenerate TypeScript API types

Run:

```bash
just typegen
just check-typegen
```

Commit the regenerated `interface/src/api/schema.d.ts`. Never hand-edit it.

### 8. Add the frontend page

Create `interface/src/pages/<Name>.tsx` modeled on `Items.tsx` or `Posts.tsx`.

Frontend conventions:

- Use the typed `api` client from `../api/client`; never raw `fetch`.
- Use `useQuery` with array query keys.
- Mutations should invalidate affected queries on success.
- Include explicit loading and error states.
- Keep styling inline with Tailwind classes in the zinc palette.
- Use spaces, not tabs.
- Do not introduce `@/` path aliases.

### 9. Add the frontend route and navigation

In `interface/src/router.tsx`:

- import the page;
- add a `createRoute({ ... })` entry;
- append it to `rootRoute.addChildren([...])`;
- add a `<Link>` in `Layout`.

## Verification matrix

Run the gates that match the change:

```bash
just lint            # Rust fmt/clippy + frontend tsc
just test            # backend tests against in-memory SQLite
just check-typegen   # verifies generated TypeScript is current
just build           # for embedded UI, release, Docker, or packaging changes
```

If you changed API annotations or schemas, run `just typegen` before
`just check-typegen` and commit the regenerated schema.

## Removing the examples

Generated apps should replace `items` and `posts` with their own domain when
the examples are no longer useful. Remove them deliberately across every layer:

- migration/data model if the app has not shipped with those tables;
- domain modules and `pub mod` exports;
- API handlers, OpenAPI paths/schemas, and routes;
- tests;
- generated TypeScript after `just typegen`;
- frontend pages and router links.

If an app has already shipped with example tables, treat removal as a real data migration, not a template cleanup.
