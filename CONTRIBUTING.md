# Contributing

## Setup and gates

See AGENTS.md for the full conventions and the step-by-step recipe for adding
a resource. Before any commit:

```
just lint            # fmt check + clippy -D warnings + tsc, same as CI
just test            # backend black-box tests
just check-typegen   # fail if committed TS types are stale (CI enforces this)
```

Use conventional commit messages (`feat:`, `fix:`, `ci:`, `docs:`, `test:`).
Keep commits small and focused.

## Hard rules

- Migrations are append-only: never edit or rename a committed file in
  `migrations/` (sqlx checksums them).
- `interface/src/api/schema.d.ts` is generated by `just typegen` and
  committed; never hand-edit it.
- JS tooling is bun/bunx only.
- No code copied from license-incompatible projects and no third-party
  product names in the template. Prefer clean-room reimplementation of
  patterns over copying files.
- `VISION.md` is human-maintained. Do not edit it in a PR unless you are a human
  maintainer intentionally revising the vision; agents and tooling must leave it
  untouched.

## Backporting from generated projects

The template improves through real apps built on it. When you fix or improve
something in a generated project that is not specific to its domain:

- Bug in template wiring, CI, build, or docs: open a PR against the template
  with the fix applied to the `items`/`posts` examples.
- Reusable pattern (a new layer, workflow, or convention): open an issue
  first describing the problem it solves and where it lives in the recipe;
  template patterns must work for both examples, not just your app.
- Anything touching defaults (security, CORS, dependencies, release): open an
  issue for discussion before a PR.

Signals that something belongs upstream: repeated friction in the recipe, a
missing abstraction you had to invent, a weak or unsafe default, release or build
pain, a security gap, platform inconsistency, a performance regression, or a
developer-experience papercut. The **Change request** issue template captures the
category and risk level so the right path (PR now, or discuss first) is obvious to
humans and AI agents alike.

Keep backports free of your app's domain code; port the pattern, not the
feature.
